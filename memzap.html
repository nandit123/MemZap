Setup

Wire the bridge in four steps

Create the memory space, point Slack at it, point Claude at it. The dashboard fills in your network endpoints automatically — you're on Testnet right now.

STEP 1

Create your memory space

Walrus Memory scopes everything to an account plus a namespace. Generate a fresh account ID and delegate key from the hosted endpoint, then paste them back here — they wire every config below.

Generate keys at Walrus Memory
The delegate key lives only in your bot server's environment. Claude never needs it — it signs in through a browser flow instead. Never paste it into a Claude config file.
STEP 2

Connect Slack

Create a Slack app from this manifest, then run the bot server. When you tag @memzap remember … in any channel, it writes that message — plus where it came from — into your memory space.

slack-app-manifest.yaml
At api.slack.com/apps → Create New App → From manifest, paste the above. Install to your workspace, then copy the bot, app, and signing tokens into the server's .env.

STEP 3

Connect Claude

Walrus Memory ships an MCP server, so Claude reads your memory with zero custom code. Drop this into your Claude config and restart. The first time Claude calls a memory tool, it opens a browser sign-in — no keys in the file.

claude_desktop_config.json
Open Claude → Settings → Developer → Edit config, paste, and restart Claude. Then ask Claude to recall — it gets the tools memwal_recall and memwal_remember against your slack namespace.
Using Cursor or Claude Code instead? Same block, different file (~/.cursor/mcp.json). The package and flags are identical.

How it works

The whole bridge, start to finish

Slack writes from one side, Claude reads from the other, and the memory you own sits in the middle. Here's each piece.

1

You own a memory space

An account on Sui plus a namespace. Content is encrypted with Seal before it touches Walrus storage — only your keys and chosen delegates can read it.

2

Slack writes through a bot

Tag the bot; its server catches the mention, captures the channel, author, and permalink, and calls memwal.remember() via the relayer.

3

Walrus stores it durably

The relayer embeds the text, encrypts it, and writes the blob to Walrus. The Sui contract records ownership so nobody else can read or alter it.

4

Claude reads through MCP

The Walrus Memory MCP server gives Claude memwal_recall. Ask a question; Claude pulls the matching memory — including ones written in Slack.

5

Origins stay attached

Every memory keeps its source. The dashboard shows which Slack channel, person, and message each one came from, with a link back.

6

Testnet first, then mainnet

Build on --staging with test endpoints. Flip one toggle to move to --prod mainnet — the configs regenerate, your flow doesn't change.
